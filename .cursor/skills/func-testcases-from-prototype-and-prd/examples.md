## 示例 A：用户管理模块（原型 + PRD）

### 输入信息

**需求文档摘要（PRD 片段）**：
- **R1**：支持新增用户，必填项：用户名（全局唯一，2-20 字符）、手机号（11 位手机号格式）、角色（下拉选择）
- **R2**：支持编辑用户信息，编辑时用户名不可修改，其余校验规则同新增
- **R3**：支持启用/禁用用户，禁用的用户不可登录
- **R4**：支持按用户名（模糊搜索）、角色（多选）、状态（多选）查询用户列表
- **R5**：查询条件支持重置
- **R6**：用户列表分页展示，默认每页 20 条
- **R7**：只有管理员角色可见"新增""编辑""启用/禁用"按钮，普通用户仅可查看列表

**原型图描述**：
- 顶部：查询区（用户名输入框、角色下拉多选、状态下拉多选、查询按钮、重置按钮）
- 中部：新增按钮（仅管理员可见）
- 列表区：表格列（用户名、手机号、角色、状态、创建时间、操作列[编辑/启用禁用]）
- 底部分页器
- 新增/编辑为弹窗形式

## 示例 B：订单审批流程（纯需求文档，无原型）

### 输入信息

**需求文档摘要**：
- **R1**：用户提交订单后，订单状态变为"待审批"
- **R2**：部门经理可审批本部门订单，审批通过后状态变为"已通过"，审批驳回后状态变为"已驳回"并需填写驳回原因
- **R3**：金额超过 10 万元的订单，部门经理审批通过后需总经理二次审批
- **R4**：审批通过/驳回后，提交人收到站内消息通知
- **R5**：提交人可在"待审批"状态下撤回订单，撤回后状态变为"草稿"
- **R6**：已驳回的订单，提交人可修改后重新提交

## 示例 C：电商下单全链路（原型 + PRD，含多分支与回退）

### 输入信息

**需求文档摘要**：
- **R1**：用户可浏览商品、加入购物车、创建订单
- **R2**：下单后 30 分钟内需完成支付，超时自动取消
- **R3**：支付成功后，系统自动分配至仓库拣货
- **R4**：发货后用户可查看物流信息
- **R5**：用户可确认收货，收货后 7 天内可申请退货退款
- **R6**：退款按原支付方式退回，优惠券不退还
- **R7**：VIP 用户享受免运费，普通用户满 99 元免运费

### 业务链路总览

```
主链路：浏览商品 → 加入购物车 → 创建订单 → 支付 → 仓库拣货 → 发货 → 确认收货 → 完成

大额分支（订单>5000）：…→ 创建订单 → 需人工审核 → 审核通过 → 支付 → …

超时中断：…→ 创建订单 → 30分钟未支付 → 自动取消 → 库存释放

退货回退：…→ 确认收货 → 申请退货 → 审核通过 → 退货寄回 → 仓库收货 → 退款到账
```
