# 校验、数据与优先级参考

## 常用校验目录

- **必填**：空 / 仅空格
- **长度**：最小/最大、多字节字符
- **格式**：邮箱/手机号/证件号、日期时间、URL、编码规则
- **范围**：数值边界、小数位、负数、0
- **唯一性**：全局唯一 vs 同层级唯一（如上级下的名称不重复）
- **安全（黑盒）**：XSS 字符串、SQL 注入类输入、文件上传 MIME 伪装
- **业务校验**：日期先后关系、金额正负、状态前置条件

## 高风险校验补充模板

- **导入模板校验**：文件类型、表头完整性、列顺序容错、空行处理、重复行识别。
- **导入数据校验**：行级错误（必填/格式/范围）与业务级错误（权限/状态/关联对象）分层提示。
- **导入结果校验**：全成功、部分成功、全失败下的数据落库一致性与回执一致性。
- **层级删除校验**：父级删除级联规则、部分可删时的拦截提示与剩余数据一致性。
- **切换校验**：切换前完整性条件、切换后数据继承（状态/标记/不可编辑字段）一致。
- **提交校验**：阻断类校验必须在提交前统一拦截；异常明细需可追溯到对象与月份/字段。
- **并发校验**：重复点击保存/提交/导入只生效一次，避免重复数据。

## 数据差异/组合

- **语言/国际化**：中/英、超长文案、RTL（若支持）
- **时间**：时区、夏令时、月末、闰日
- **金额/数字**：千分位、四舍五入/截断规则

## 兼容性 / 可访问性（仅在需求或原型显著相关时覆盖）

- **兼容**：Chrome/Edge/Safari、iOS/Android 版本、小屏适配
- **可访问性**：纯键盘可用、焦点顺序、读屏标签、对比度

## 优先级建议

- **P0**：核心业务不可用、数据丢失/错乱、安全/资金/订单/权限问题
- **P1**：重要功能明显降级或状态错误，但存在绕行方案；或列表辅助能力（查询、重置、日志、权限、备注等）
- **P2**：轻微 UI/文案、低频边界、非阻塞问题
