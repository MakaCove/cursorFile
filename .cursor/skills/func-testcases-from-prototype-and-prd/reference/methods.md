# 测试用例设计八大方法

生成测试用例时**最基本要遵守**以下八大规则；根据功能与需求选择适用方法，使用例覆盖充分且不冗余。

## 1. 等价类划分法

将输入域划分为若干**等价类**，同一等价类内数据在程序处理上等价，从每类中选取**代表值**测试即可，避免重复。
- **有效等价类**：合法、符合规格的输入，每类至少 1 条用例。
- **无效等价类**：非法、不符合规格的输入（如格式错误、越权），每类至少 1 条用例。
适用于：必填/选填、格式（手机号/邮箱）、枚举（下拉选项）、范围（金额/数量）等。

## 2. 边界值分析法

在输入范围的**边界**及**边界外侧**设计用例，边界处易产生缺陷。
- 典型取点：最小值、最大值、刚好小于最小值、刚好大于最大值、空/零。
- 长度类：最小长度、最大长度、超长、空串。
- 数量类：0 条、1 条、满页、超页。
常与等价类结合：先划分等价类，再对边界类做边界值分析。

## 3. 场景法（流程法）

按用户**真实使用场景/业务流程**设计用例，覆盖端到端路径。
- **主场景（Happy path）**：一条从入口到目标完成的完整成功路径。
- **备选/分支场景**：取消、返回、失败重试、多 Tab/多入口。
- **异常场景**：网络异常、超时、服务端错误、权限不足。
适用于：下单、审批、配置保存、导入导出等有明确流程的功能。

## 4. 错误推测法

基于**经验**推测易错点，补充针对性用例。
- 常见易错：空值/空格、重复提交、重复数据、权限越界、未登录、网络断开、超时、并发重复点击、特殊字符、超长输入。
- 业务易错：状态不允许时操作、已删除/已禁用对象被引用、数据被他人修改后的并发保存。
与等价类、边界值互补，用于查漏。

## 5. 因果图法

分析**输入条件（因）**与**输出/结果（果）**的因果关系及约束（互斥、包含、唯一等），避免遗漏逻辑组合。
- 步骤：列出输入条件与输出结果 → 画因果图 → 转为判定表或直接写出用例。
适用于：多条件共同决定一个结果（如"有 A 且无 B 时才允许 C"）、条件间有约束时。

## 6. 判定表法

将**多个条件的组合**与**对应动作/结果**列成**判定表**，每列（或每行）代表一条规则，保证条件组合覆盖完整。
- 条件桩：所有参与判断的条件。
- 动作桩：可能的结果或操作。
- 条件项与动作项：每一列为一组条件取值及对应动作。
适用于：多条件组合（如筛选条件、权限×操作、业务规则组合），可先写判定表再转为用例。

## 7. 正交试验法

当**因素（输入维度）多、水平（取值）多**时，用**正交表**选取有代表性的组合，用较少用例覆盖主要组合。
- 目的：减少用例数量，同时保证关键组合被覆盖。
- 适用：多下拉框、多配置项、多环境参数等组合测试；不必穷举，选正交代表即可。

## 8. 状态迁移法

针对有**明确状态**的对象（如待提交、已提交、已审核、已驳回），画出**状态图**，覆盖：
- **合法迁移**：从某状态到下一状态的正向路径。
- **非法迁移**：不允许的迁移（如已审核后再编辑）应被拒绝并有提示。
适用于：启用/禁用、草稿/已提交/已审核、订单/工单状态、审批流等。
