# 参考：覆盖维度与边界用例

本页为**可选参考**：根据截图与需求发散思考时，可从下列维度中选取与当前页面相关的项，不要求全部覆盖。

## 测试用例设计八大方法（描述）

生成测试用例时**最基本要遵守**以下八大规则；根据页面与需求选择适用方法，使用例覆盖充分且不冗余。

### 1. 等价类划分法

将输入域划分为若干**等价类**，同一等价类内数据在程序处理上等价，从每类中选取**代表值**测试即可，避免重复。  
- **有效等价类**：合法、符合规格的输入，每类至少 1 条用例。  
- **无效等价类**：非法、不符合规格的输入（如格式错误、越权），每类至少 1 条用例。  
适用于：必填/选填、格式（手机号/邮箱）、枚举（下拉选项）、范围（金额/数量）等。

### 2. 边界值分析法

在输入范围的**边界**及**边界外侧**设计用例，边界处易产生缺陷。  
- 典型取点：最小值、最大值、刚好小于最小值、刚好大于最大值、空/零。  
- 长度类：最小长度、最大长度、超长、空串。  
- 数量类：0 条、1 条、满页、超页。  
常与等价类结合：先划分等价类，再对边界类做边界值分析。

### 3. 场景法（流程法）

按用户**真实使用场景/业务流程**设计用例，覆盖端到端路径。  
- **主场景（Happy path）**：一条从入口到目标完成的完整成功路径。  
- **备选/分支场景**：取消、返回、失败重试、多 Tab/多入口。  
- **异常场景**：网络异常、超时、服务端错误、权限不足。  
适用于：下单、审批、配置保存、导入导出等有明确流程的功能。

### 4. 错误推测法

基于**经验**推测易错点，补充针对性用例。  
- 常见易错：空值/空格、重复提交、重复数据、权限越界、未登录、网络断开、超时、并发重复点击、特殊字符、超长输入。  
- 业务易错：状态不允许时操作、已删除/已禁用对象被引用、数据被他人修改后的并发保存。  
与等价类、边界值互补，用于查漏。

### 5. 因果图法

分析**输入条件（因）**与**输出/结果（果）**的因果关系及约束（互斥、包含、唯一等），避免遗漏逻辑组合。  
- 步骤：列出输入条件与输出结果 → 画因果图 → 转为判定表或直接写出用例。  
适用于：多条件共同决定一个结果（如“有 A 且无 B 时才允许 C”）、条件间有约束时。

### 6. 判定表法

将**多个条件的组合**与**对应动作/结果**列成**判定表**，每列（或每行）代表一条规则，保证条件组合覆盖完整。  
- 条件桩：所有参与判断的条件。  
- 动作桩：可能的结果或操作。  
- 条件项与动作项：每一列为一组条件取值及对应动作。  
适用于：多条件组合（如筛选条件、权限×操作、业务规则组合），可先写判定表再转为用例。

### 7. 正交试验法

当**因素（输入维度）多、水平（取值）多**时，用**正交表**选取有代表性的组合，用较少用例覆盖主要组合。  
- 目的：减少用例数量，同时保证关键组合被覆盖。  
- 适用：多下拉框、多配置项、多环境参数等组合测试；不必穷举，选正交代表即可。

### 8. 状态迁移法

针对有**明确状态**的对象（如待提交、已提交、已审核、已驳回），画出**状态图**，覆盖：  
- **合法迁移**：从某状态到下一状态的正向路径。  
- **非法迁移**：不允许的迁移（如已审核后再编辑）应被拒绝并有提示。  
适用于：启用/禁用、草稿/已提交/已审核、订单/工单状态、审批流等。

## 截图解读清单（UI 维度）

- **导航**：面包屑、返回行为、路由参数、深链
- **搜索/筛选**：筛选条件（多选/单选/文本+模糊搜索）、**查询**与**重置**按钮、默认值、筛选标签、输入防抖
- **列表/表格**：分页、排序、列显隐/宽度、行选择、批量操作、**备注列及备注编辑入口**
- **表单**：必填标识、placeholder vs label、错误提示位置、帮助文案、自动保存
- **弹窗/抽屉**：关闭按钮、ESC、点击遮罩、未保存提示、焦点陷阱
- **Toast/提示**：时机、堆叠、重复提交去重
- **权限**：隐藏/禁用/点击后报错的策略一致性；**接口无权限时的返回**
- **空状态**：首次进入、筛选后无结果
- **加载**：骨架屏/转圈、重复刷新、取消请求
- **错误**：重试、离线、超时、服务端 4xx/5xx、部分失败

## 常见覆盖维度（按页面与需求发散选用）

以下为常见能力维度，**仅当截图或需求中确实存在时**才纳入用例设计，不必全部覆盖：

- **列表筛选/查询**：筛选条件（多选、单选、文本、模糊搜索等）→ 点击查询 → 列表展示符合条件数据（条件关系以需求为准，如并集/交集）
- **重置**：点击重置 → 筛选条件清空，列表刷新（具体表现以本页为准）
- **操作日志/审计**：若需求或 UI 有“操作日志”，可覆盖：哪些操作写入日志、日志展示顺序与内容
- **功能权限**：若需求提到“按菜单/按钮控制”，可覆盖：无权限时按钮可见性/可用性及提示
- **接口权限**：若需求提到接口鉴权，可覆盖：直接调接口时无权限的返回信息
- **行内编辑/备注**：若表格或详情有备注、行内编辑等入口，可覆盖：编辑、保存、列表展示与可编辑频率
- **批量/二次确认**：若有启用禁用、删除等，可覆盖：单个/批量、确认/取消、状态校验与“跳过不符合条件项”等规则
- **配置类页面（多块 + 行动态增删）**：若页面为“配置块 + 每块多行 + 行尾添加/删除”，需覆盖：每块必填项与默认值、行的添加与删除、**仅剩一条时删除置灰或不可删**（边界）、块内或跨行**唯一性校验**（如“某字段在同一块内不能重复”）、保存后配置生效范围（如“仅影响新生成数据”）
- **报表/看板类页面**：若页面为“根据配置与源数据生成的报表/看板”，需覆盖：**数据来源与展示规则**（来自某配置/某表、展示名称与排序）、**树形/层级展示**（默认展开/收缩、一级与合并行样式如浅红+红竖线）、**多维度分行规则**（如末级按科目+辅助项+公司分行）、**字段计算公式**（如简单加计=各公司之和、合并汇总=简单加计+借方调整-贷方调整）、**自动更新 vs 手动更新**（源数据变更自动更新、配置变更需手动更新）、**手动更新弹窗**（主体账簿多选/必填/模糊搜索、会计期间必填、必填校验文案）、**导出**（导出当前查询全部、导出文件名含报表名+期间）、**数据权限**（公司/主体账簿按权限展示列与数据）、不翻页时用“导出当前查询”覆盖数据范围

## 常用校验目录（按需选择）

- **必填**：空 / 仅空格
- **长度**：最小/最大、多字节字符
- **格式**：邮箱/手机号/证件号、日期时间、URL、编码规则
- **范围**：数值边界、小数位、负数、0
- **唯一性**：全局唯一 vs 同层级唯一（如上级下的名称不重复）、与其它实体（如会计科目）不重复
- **安全（黑盒）**：XSS 字符串、类似 SQLi 的输入、文件上传 MIME 伪装

## 数据差异/组合

- **语言/国际化**：中/英、超长文案、RTL（若支持）
- **时间**：时区、夏令时、月末、闰日
- **金额/数字**：千分位、四舍五入/截断规则

## 兼容性 / 可访问性（仅在需求或页面显著相关时覆盖）

- **兼容**：Chrome/Edge/Safari、iOS/Android 版本、小屏适配
- **可访问性**：纯键盘可用、焦点顺序、读屏标签、对比度

## 优先级建议

- **P0**：核心业务不可用、数据丢失/错乱、安全/资金/订单/权限问题
- **P1**：重要功能明显降级或状态错误，但存在绕行方案；或列表辅助能力（查询、重置、日志、权限、备注等）
- **P2**：轻微 UI/文案、低频边界、非阻塞问题
